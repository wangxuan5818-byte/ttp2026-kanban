# TTP2026 战略看板系统技术报告 (v2.0)

## 1. 架构概述

TTP2026 战略看板系统 (v2.0) 经历了一次全面的架构重构，从原有的 Node.js + tRPC + SQLite 架构，平滑迁移至 **Python (FastAPI) + MySQL + React + Electron** 的现代化全栈架构。本次重构旨在提升系统的并发处理能力、数据存储的可靠性以及跨平台部署的灵活性，同时保持了前端业务逻辑的稳定性。

### 1.1 核心技术栈

*   **前端 (Web端)**: React 18, Vite, TailwindCSS, tRPC Client, Radix UI
*   **客户端 (桌面端)**: Electron 28, Electron-Builder
*   **后端 (API服务)**: Python 3.11, FastAPI, Uvicorn, Pydantic
*   **数据库**: MySQL 8.0 / MariaDB, SQLAlchemy (ORM), PyMySQL
*   **认证授权**: JWT (JSON Web Tokens), passlib (sha256_crypt)

### 1.2 架构演进亮点

本次架构升级最大的亮点在于**无缝迁移**。为了避免大规模重写前端代码带来的风险和成本，我们在 Python 后端实现了一个创新的 **tRPC 兼容层** (`trpc_compat.py`)。该兼容层能够解析前端发出的 tRPC 批量请求格式，将其路由到对应的 Python 处理函数，并将结果按照 tRPC 期望的格式返回。这使得前端代码几乎零修改即可对接全新的 Python 后端。

## 2. 系统模块设计

### 2.1 前端展示层 (React + Vite)

前端采用 React 构建，使用 Vite 作为构建工具。UI 组件库基于 Radix UI 和 TailwindCSS，保证了界面的现代化和响应式。

*   **状态管理**: 使用 `@tanstack/react-query` 配合 tRPC 客户端进行服务端状态管理和数据缓存。
*   **核心组件**:
    *   `KanbanBoard.tsx`: 核心看板视图，展示各部门任务卡片。
    *   `Sidebar.tsx`: 左侧导航栏，v2.0 版本重构为平铺展示所有 13 个部门（包含新增的参谋部和政治部）。
    *   `OutcomePanel.tsx` & `GlobalOutcomePanel.tsx`: 效益核算模块，提供 ROI 仪表盘、场景对比分析等复杂数据可视化。
    *   `GlobalEditor.tsx`: 全局数据编辑器，供管理员管理部门和任务。

### 2.2 桌面客户端层 (Electron)

为了提供更好的沉浸式体验，系统新增了 Electron 桌面客户端。

*   **主进程 (`main.js`)**: 负责创建应用窗口、管理应用生命周期。启动时会展示一个定制的 TTP2026 启动画面（Splash Screen），提升品牌感。
*   **集成方式**: Electron 作为一个外壳，加载本地运行的 Vite 前端服务或打包后的静态文件，实现 Web 技术的桌面化。

### 2.3 后端服务层 (FastAPI)

后端完全使用 Python 重写，基于高性能的 FastAPI 框架。

*   **路由设计**:
    *   `/api/auth`: 处理登录、登出、Token 刷新。
    *   `/api/tasks`: 任务的 CRUD 操作。
    *   `/api/outcomes`: 效益核算数据的管理与汇总计算。
    *   `/api/trpc`: **核心兼容层**，处理来自前端的 tRPC 批量请求。
*   **数据验证**: 广泛使用 Pydantic (`schemas.py`) 进行请求和响应数据的强类型验证，确保数据安全。
*   **密码安全**: 考虑到跨平台兼容性，放弃了容易产生依赖问题的 bcrypt，转而使用 `passlib` 提供的 `sha256_crypt` 算法进行密码哈希存储。

### 2.4 数据持久层 (MySQL + SQLAlchemy)

数据库从轻量级的 SQLite 迁移至企业级的 MySQL，使用 SQLAlchemy 作为 ORM 工具。

*   **核心表结构**:
    *   `users`: 存储用户账号、密码哈希、角色及所属部门。包含 14 个预设账号（含 admin, canmoubu, zhengzhibu 等）。
    *   `tasks`: 存储战略任务详情，包括目标、策略、里程碑、状态等。
    *   `outcomes`: 存储效益核算记录，支持提效、降本、增收三种类型，记录改善前后的数值。
    *   `scores`: 存储用户积分记录。
    *   `committee_configs`: 存储部门的配置信息（如年度目标、奖金包等）。

## 3. 核心业务流分析

### 3.1 tRPC 兼容层数据流

这是新架构中最关键的数据流：

1.  **前端发起请求**: 前端 React 组件通过 tRPC 客户端发起请求（例如 `kanban.login` 或 `tasks.list`）。由于配置了批量请求（batching），多个请求可能被合并为一个 HTTP POST 请求发送到 `/api/trpc/{routes}`。
2.  **FastAPI 接收**: 后端的 `trpc_compat.py` 路由捕获该请求。
3.  **解析与路由**: 兼容层解析 URL 中的路由名称和 Body 中的输入数据（通常包含在 `0` 这样的键下），并根据路由名称分发到具体的处理函数（如 `handle_kanban_login`）。
4.  **业务处理**: 处理函数调用 SQLAlchemy 与 MySQL 交互，执行业务逻辑。
5.  **格式化响应**: 兼容层将处理结果包装成 tRPC 期望的 JSON 格式（例如 `[{"result": {"data": {...}}}]`）并返回给前端。

### 3.2 效益核算数据流

效益核算是 v2.0 的重点功能：

1.  **数据录入**: 用户在 `OutcomePanel` 中录入改善前后的数值（如工时、成本、收入）。
2.  **后端计算**: FastAPI 接收到数据后，存入 `outcomes` 表。
3.  **全局汇总**: 当管理员查看 `GlobalOutcomePanel` 时，前端发起 `outcomes.globalSummary` 请求。
4.  **聚合查询**: 后端执行 SQL 聚合查询，按部门分组计算总提效、总降本、总增收，并计算年化总价值和平均 ROI。
5.  **前端渲染**: 前端接收汇总数据，使用图表组件渲染 ROI 仪表盘和部门对比图。

## 4. 部署与运维

新架构支持灵活的部署方式：

*   **一键启动**: 提供了 `start.sh` 和 `start.bat` 脚本，可一键启动 MySQL 服务、FastAPI 后端和 Vite 前端。
*   **生产环境部署**:
    *   **数据库**: 推荐使用独立的 MySQL 8.0 实例。
    *   **后端**: 使用 Gunicorn 管理 Uvicorn worker 进程（如 `gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker`），并使用 PM2 或 Systemd 守护进程。
    *   **前端**: 运行 `pnpm build` 生成静态文件，使用 Nginx 代理静态文件，并将 `/api` 路径反向代理至 FastAPI 后端。
*   **客户端打包**: 使用 `npm run build:win` 或 `npm run build:mac` 通过 electron-builder 将应用打包为独立的安装程序。

## 5. 总结

TTP2026 战略看板 v2.0 的架构重构是一次成功的技术升级。通过引入 Python 和 MySQL，系统获得了更好的扩展性和数据处理能力；通过 Electron，系统实现了跨平台的桌面端覆盖；而创新的 tRPC 兼容层设计，则以最小的代价实现了前后端的平滑对接。这一新架构为 TTP 集团未来的战略管理提供了坚实的技术底座。
