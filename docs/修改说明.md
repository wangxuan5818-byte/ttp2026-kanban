## 五、效益核算板块详细开发

**文件：** `client/src/components/OutcomePanel.tsx`（约950行，完全重写）

新增功能：

| 功能模块 | 说明 |
|---------|------|
| **ROI仪表盘** | 年化总价值、平均ROI、效益等级（S/A/B/C/D）、三类效益迷你图 |
| **效益等级评估** | S(≥500%) / A(200-499%) / B(100-199%) / C(50-99%) / D(<50%) |
| **场景对比分析** | AI前后可视化对比，改善幅度百分比 |
| **效益记录卡片** | 详细展示每条记录，支持编辑/删除 |
| **双视图切换** | 记录视图 + 分析视图 |
| **新增记录表单** | 完整的效益录入表单，含场景、时间节省、质量提升、成本降低 |

**ROI计算公式：**

```
年化价值（提效）= (AI前耗时 - AI后耗时) × 月频次 × 12 × 300元/小时
年化价值（降本）= (AI前成本 - AI后成本) × 12
年化价值（增收）= (AI后收入 - AI前收入) × 频次

ROI = 年化产出价值 ÷ 投入成本 × 100%
投入成本 = 投入工时 × 800元/人天
```

**文件：** `client/src/components/GlobalOutcomePanel.tsx`（新建）

- 管理员总览页展示各部门效益排行
- 显示各部门年化价值、ROI、效益等级
- 集成到 `OverviewPanel.tsx` 中

## 六、后端重构（Node.js → Python FastAPI）

**新建目录：** `backend/`

| 文件 | 说明 |
|------|------|
| `backend/main.py` | FastAPI应用入口，注册所有路由 |
| `backend/models/database.py` | SQLAlchemy ORM数据模型 |
| `backend/schemas/schemas.py` | Pydantic请求/响应模式 |
| `backend/routers/auth.py` | JWT认证路由 |
| `backend/routers/tasks.py` | 任务CRUD路由 |
| `backend/routers/outcomes.py` | 效益核算路由 |
| `backend/routers/committees.py` | 部门配置路由 |
| `backend/routers/users.py` | 用户管理路由 |
| `backend/routers/trpc_compat.py` | **tRPC兼容层**（前端无需修改） |
| `backend/init_db.sql` | 数据库初始化SQL（含14个默认账号） |
| `backend/.env.example` | 环境变量模板 |

**tRPC兼容层说明：**

原前端使用tRPC协议（`/api/trpc/*`），Python后端通过 `trpc_compat.py` 实现兼容，将tRPC批量请求转换为REST调用，前端代码**无需任何修改**即可使用Python后端。
