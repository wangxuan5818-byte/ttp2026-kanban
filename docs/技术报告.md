# TTP2026 战略看板系统架构重构技术报告

## 1. 项目背景与目标

TTP2026 战略看板系统原基于 Node.js 和 tRPC 架构开发。为了更好地支持后续的数据分析、AI 模型集成以及更稳定的企业级部署，本项目对系统架构进行了全面重构。

**重构目标：**
1. 将后端从 Node.js 迁移至 Python FastAPI。
2. 引入 MySQL 关系型数据库替换原有的数据存储方案。
3. 保持前端 React 架构不变，通过 tRPC 兼容层实现无缝对接。
4. 扩展系统功能，新增参谋部和政治部模块。
5. 完善效益核算板块，提供详细的 ROI 分析。
6. 支持 Web 端和 Electron 桌面客户端双端访问。

## 2. 架构设计

### 2.1 整体架构

重构后的系统采用前后端分离架构，主要技术栈如下：

*   **前端：** React 18 + TypeScript + Vite + TailwindCSS + Recharts
*   **后端：** Python 3.x + FastAPI + SQLAlchemy + PyJWT + bcrypt
*   **数据库：** MySQL 8.0
*   **桌面端：** Electron 28.3.3

### 2.2 后端架构 (Python FastAPI)

后端采用 FastAPI 框架，利用其高性能和自动生成 API 文档的特性。

*   **路由层 (Routers)：** 处理 HTTP 请求，包括标准的 RESTful API 和 tRPC 兼容接口。
*   **业务逻辑层 (Services)：** 处理核心业务逻辑，如用户认证、任务管理、数据统计等。
*   **数据访问层 (DAL)：** 使用 SQLAlchemy ORM 与 MySQL 数据库进行交互。
*   **模型层 (Models)：** 定义数据库表结构（User, Task, Milestone, Outcome 等）。

### 2.3 tRPC 兼容层设计

为了避免大规模修改前端代码，后端实现了一个 tRPC 兼容层。前端依然使用 tRPC 客户端发起请求，后端 FastAPI 接收到 `/trpc/{path}` 的请求后，解析请求体，并将其路由到对应的 Python 处理函数。

**实现细节：**
*   解析 tRPC 的批量请求格式（Batch Requests）。
*   将 tRPC 的输入参数映射为 Python 函数的参数。
*   将 Python 函数的返回值格式化为 tRPC 客户端期望的响应格式。

### 2.4 数据库设计 (MySQL)

系统采用 MySQL 8.0 作为持久化存储，主要表结构包括：

*   `users`: 存储用户信息、密码哈希、角色和权限范围。
*   `tasks`: 存储战略任务信息，包括所属部门、状态、进度等。
*   `milestones`: 存储任务的里程碑信息。
*   `outcomes`: 存储效益核算相关数据。

## 3. 核心功能实现

### 3.1 部门模块扩展

系统在原有的 11 个部门基础上，新增了**参谋部**和**政治部**。

*   **参谋部：** 核心任务包括宣发推广、引流、策划方案、AI 数字人宣传。
*   **政治部：** 核心任务包括培训课程认证、三体人教学创建。

前端侧边栏组件 (`Sidebar.tsx`) 进行了优化，取消了分组，改为平铺显示 13 个部门，提升了导航效率。

### 3.2 效益核算板块

效益核算板块 (`OutcomePanel.tsx`) 进行了深度开发，提供了丰富的数据可视化功能：

*   **ROI 仪表盘：** 实时展示各部门的投资回报率。
*   **场景对比分析：** 对比不同业务场景的效益指标。
*   **效益等级评估：** 根据预设规则对任务效益进行自动评级。

### 3.3 数据导入与初始化

系统支持从 Excel 表单导入初始数据。通过解析 "AI战略会-TTP2026.xlsx"，成功将 26 个核心任务导入到各个部门。同时，提供了数据库初始化脚本 (`init_db.py`)，自动创建 14 个默认账号和基础数据。

### 3.4 任务状态与战略路径更新

*   将 2025 年 12 月的 27 个任务全部标记为 "已结束" 状态。
*   月度战略路径扩展至 2026 年 05 月，新增了 AI 人才培养、跨行业扩展、ChatYE 平台、创作者平台等内容。

## 4. 部署与打包

系统支持 Web 端和桌面端两种部署方式。

*   **Web 端：** 前端使用 Vite 构建静态文件，后端使用 Uvicorn 运行 FastAPI 服务。
*   **桌面端：** 使用 Electron 封装前端页面，并通过 `electron-builder` 打包为可执行文件。
*   **后端打包：** 使用 PyInstaller 将 Python 后端打包为独立的可执行文件，简化部署流程。

## 5. 总结

本次架构重构成功将 TTP2026 战略看板系统迁移至 Python + MySQL 技术栈，不仅提升了系统的稳定性和可扩展性，还通过 tRPC 兼容层实现了前端的平滑过渡。新增的部门模块和完善的效益核算板块进一步丰富了系统功能，为企业的战略决策提供了有力的数据支持。
