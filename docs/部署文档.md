# TTP2026 战略看板 · 部署文档 v2.0

> **技术栈：React（Vite）+ Python FastAPI + MySQL + Electron**
> 适用环境：Ubuntu 22.04 / CentOS 7+ / Windows Server 2019+

---

## 二、快速启动（开发环境）

### 2.1 一键启动（Linux/Mac）

```bash
cd /path/to/ttp2026-kanban
chmod +x start.sh
./start.sh
```

### 2.2 一键启动（Windows）

```bat
cd C:\path\to\ttp2026-kanban
start.bat
```

### 2.3 手动启动步骤

**步骤1：启动MySQL并创建数据库**

```bash
sudo mysql -u root
```

```sql
CREATE DATABASE IF NOT EXISTS ttp2026_kanban CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'kanban'@'localhost' IDENTIFIED BY 'kanban2026';
GRANT ALL PRIVILEGES ON ttp2026_kanban.* TO 'kanban'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

**步骤2：安装Python依赖**

```bash
cd backend
pip3 install fastapi uvicorn sqlalchemy pymysql python-jose passlib python-dotenv cryptography
```

**步骤3：配置环境变量**

```bash
cp backend/.env.example backend/.env
# 编辑 backend/.env
```

`.env` 文件内容：

```env
DATABASE_URL=mysql+pymysql://kanban:kanban2026@localhost/ttp2026_kanban
SECRET_KEY=ttp2026-secret-key-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=480
```

**步骤4：初始化数据库**

```bash
# 方式一：执行SQL文件
sudo mysql -u root ttp2026_kanban < backend/init_db.sql

# 方式二：运行Python初始化脚本
cd backend && python3 -c "from models.database import Base, engine; Base.metadata.create_all(engine)"
```

**步骤5：启动Python后端**

```bash
cd backend
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

后端启动后访问 `http://localhost:8000/docs` 查看Swagger API文档。

**步骤6：安装前端依赖并启动**

```bash
# 在项目根目录
npm install
NODE_ENV=development npx vite --port 3000 --host 0.0.0.0
```

**步骤7：访问系统**

浏览器打开：`http://localhost:3000`

## 三、生产环境部署

### 3.1 前端构建

```bash
# 在项目根目录
NODE_ENV=production npx vite build
# 构建产物在 client/dist/
```

### 3.2 Python后端生产部署

**方式一：PM2管理（推荐）**

```bash
# 安装PM2
npm install -g pm2

# 创建PM2配置文件 ecosystem.config.js
cat > ecosystem.config.js << 'EOF'
module.exports = {
  apps: [{
    name: 'ttp2026-backend',
    script: 'python3',
    args: '-m uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4',
    cwd: './backend',
    env: { NODE_ENV: 'production' },
    error_file: '/var/log/ttp2026/backend-error.log',
    out_file: '/var/log/ttp2026/backend-out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss',
  }]
};
EOF

mkdir -p /var/log/ttp2026
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

**方式二：systemd服务**

```bash
sudo tee /etc/systemd/system/ttp2026-backend.service << 'EOF'
[Unit]
Description=TTP2026 战略看板 Python后端
After=network.target mysql.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/ttp2026-kanban/backend
ExecStart=/usr/bin/python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable ttp2026-backend
sudo systemctl start ttp2026-backend
sudo systemctl status ttp2026-backend
```

### 3.3 Nginx配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /opt/ttp2026-kanban/client/dist;
    index index.html;

    # 前端SPA路由回退
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理到Python后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源长期缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/ttp2026 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3.4 HTTPS（Let's Encrypt）

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 四、MySQL生产配置

### 4.1 数据库创建

```sql
CREATE DATABASE IF NOT EXISTS ttp2026_kanban CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'kanban'@'localhost' IDENTIFIED BY 'strong-password-here';
GRANT SELECT, INSERT, UPDATE, DELETE ON ttp2026_kanban.* TO 'kanban'@'localhost';
FLUSH PRIVILEGES;
```

### 4.2 自动备份

```bash
cat > /opt/backup-kanban.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/opt/backups/ttp2026
mkdir -p $BACKUP_DIR
mysqldump -u kanban -pkanban2026 ttp2026_kanban > $BACKUP_DIR/kanban_$DATE.sql
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
echo "备份完成: $BACKUP_DIR/kanban_$DATE.sql"
EOF

chmod +x /opt/backup-kanban.sh
# 每天凌晨2点自动备份
echo "0 2 * * * /opt/backup-kanban.sh" | crontab -
```

## 五、Electron桌面客户端打包

### 5.1 打包前准备

```bash
# 先构建前端
NODE_ENV=production npx vite build

# 进入electron目录安装依赖
cd electron
npm install
```

### 5.2 打包命令

```bash
# 打包当前平台
npm run build

# 打包Windows（.exe安装包）
npm run build:win

# 打包Mac（.dmg）
npm run build:mac

# 打包Linux（.AppImage）
npm run build:linux
```

打包产物在 `electron/dist/` 目录。

### 5.3 客户端使用说明

1. 安装并运行Electron客户端
2. 客户端自动检测本地后端服务（端口8000）
3. 若后端未启动，会提示错误并显示启动命令
4. **启动顺序**：先启动后端 → 再启动前端 → 最后启动Electron

```bash
# 1. 启动后端
cd backend && python3 -m uvicorn main:app --port 8000 &

# 2. 启动前端（开发模式）
npx vite --port 3000 &

# 3. 启动Electron
cd electron && npm start
```

## 六、Docker部署（可选）

### 6.1 docker-compose.yml

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root2026
      MYSQL_DATABASE: ttp2026_kanban
      MYSQL_USER: kanban
      MYSQL_PASSWORD: kanban2026
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"

  backend:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: mysql+pymysql://kanban:kanban2026@mysql/ttp2026_kanban
    depends_on:
      - mysql

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./client/dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend

volumes:
  mysql_data:
```

### 6.2 Dockerfile（后端）

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .
EXPOSE 8000
CMD ["python3", "-m", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

启动：

```bash
docker-compose up -d
```

## 七、故障排查

### 7.1 后端无法启动

```bash
# 查看日志
cat /tmp/backend.log

# 检查端口占用
ss -tlnp | grep 8000

# 测试数据库连接
python3 -c "import pymysql; pymysql.connect(host='localhost', user='kanban', password='kanban2026', database='ttp2026_kanban'); print('连接成功')"
```

### 7.2 前端无法访问API

检查 `vite.config.ts` 中的代理配置：

```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    }
  }
}
```

### 7.3 登录失败

```bash
# 检查数据库用户
sudo mysql -u root -e "SELECT username, role FROM ttp2026_kanban.users;"

# 重新初始化默认数据
sudo mysql -u root ttp2026_kanban < backend/init_db.sql
```

### 7.4 tRPC调用失败

```bash
# 测试tRPC兼容层
curl -s -X POST http://localhost:8000/api/trpc/kanban.login \
  -H "Content-Type: application/json" \
  -d '{"json":{"username":"admin","password":"admin2026"}}'
```

## 八、项目目录结构

```

│   ├── main.py                 # 应用入口
```
